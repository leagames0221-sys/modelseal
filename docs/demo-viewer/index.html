「仕込み済み・出所不明・黙って変わった」モデルを堰き止めます。

non-executing x-ray + provenance + approval baseline · runtime dependencies = 0

THE PROBLEMモデルファイルは開くだけで危ないことがある

広く使われる保存形式には、読み込んだ瞬間にコードが動くものがあります。

  • 悪意あるモデルの配布は、実際に起きている攻撃手口です
  • 標準的な検査機そのものを、すり抜ける手口も公開されました
  • 安全な形式のふりをした偽物や、承認後に黙って差し替わる事故もあります

既存の検査機は「仕込みの検出」に集中しており、出どころの照合と変更の監視は空白でした。

① X-RAY実行せずに透視する非実行・読み込みゼロ

ファイルの中の命令の並びを、一度も実行せずに解析します。危険な呼び出しの気配があれば不合格。

$ modelseal scan downloaded_model.pkl
verdict: FAIL (2 findings)
  [FAIL] MS-PICKLE-DANGEROUS-IMPORT downloaded_model.pkl@2
         references a known code-execution sink
$ echo $? # 終了コード = 2 なので CI も止まる
2

② PROVENANCE出どころを照合する

指紋（ハッシュ）を計算し、配布元の公表値と照合。照合できないものを黙って通しません。

$ modelseal scan model.safetensors --expect-hash model.safetensors=9f3b...
  [INFO] MS-PROVENANCE-OK — 公表値と一致

$ modelseal scan unknown_origin.pt # 期待値なしの場合
  [WARN] MS-PROVENANCE-UNVERIFIED — 来歴未検証と明示（暗黙の合格なし）

③ BASELINE承認した中身が黙って変わってないか

レビューして承認した時点の姿を記録。以後の検査で差分を監視します。

検出意味判定
CHANGED承認後に中身が変わった警告
SEVERITY-UP変わった上に危険度が上がった無条件で不合格
ADDED / REMOVED承認外のファイルが出入りした警告
$ modelseal approve models/ --manifest baseline.json
approved 3 artifact(s) -> baseline.json
$ modelseal diff models/ --manifest baseline.json
  [FAIL] MS-BASELINE-SEVERITY-UP — risk rose since approval (INFO -> FAIL)

FORMAT INTEGRITY安全な形式のなりすましを見抜く

形式の判定は拡張子でなく中身そのもので行います。宣言と実体が食い違えば不合格。

拡張子偽装

安全形式の名前を着た実行可能コンテナ → MS-EXTENSION-SPOOF

ヘッダ不整合

宣言されたサイズや構造が実体と矛盾 → MS-SAFETENSORS-MALFORMED

BYPASS RESISTANCE検査を妨害されたら不合格に倒す

先行の検査機は「解析に失敗すると素通り」する穴を突かれました。modelseal は逆に倒します。

公開済みのすり抜け手口modelseal の応答
壊したファイルで検査機を落とすMS-UNPARSEABLE 解析不能 = 不合格
安全形式の拡張子に偽装するMS-EXTENSION-SPOOF 中身で判定
遠回しな呼び出しで検出語を避けるMS-PICKLE-DANGEROUS-IMPORT 組立経路も追跡

公開済み 3 手口は再現テストとして恒久化済み3 / 3 検出を確認

DESIGN合否判定にエーアイを関与させない

決定論コア

X 線・照合・差分の全判定。エーアイ不使用・ネット不要・毎回同じ結果。

エーアイ任意層

検出結果の平易な説明だけを担当。合否や終了コードは一切変えません。

安全の判定は決定論で確実に。説明の文章だけをローカルのエーアイが補助します。
